# **UED Asset Hub 管理与上传流程优化清单**
请基于当前页面继续优化，不要重构整体视觉风格。本次重点修正管理入口、上传表单、登录逻辑、列表管理操作。

---

## **一、管理入口逻辑调整**
### **当前问题**
现在“管理员密码”直接出现在上传页面中，逻辑不合理。

### **调整要求**
改为：

```plain
点击顶部「管理入口」
↓
进入登录页
↓
输入预设账号密码
↓
登录成功
↓
进入管理页面
↓
支持新增 / 编辑 / 删除资产
```

---

## **二、登录页新增**
新增页面：

```plain
/admin/login
```

登录页包含：

+ 账号
+ 密码
+ 登录按钮
+ 错误提示

### **预设账号**
第一期可以先写在 mock 配置中，例如：

```plain
username: admin
password: admin123
```

后续预留真实后端登录能力。

不要把管理员密码放在资产上传表单中。

---

## **三、登录态处理**
第一期可以使用 localStorage 模拟登录态。

例如：

```plain
localStorage.setItem('ued_admin_token', 'mock-token')
```

访问管理页时判断：

```plain
未登录 → 跳转 /admin/login
已登录 → 进入 /admin
```

后续需要预留真实 token / session 接入能力。

---

## **四、管理入口按钮调整**
顶部「管理入口」点击后：

未登录：

```plain
跳转 /admin/login
```

已登录：

```plain
跳转 /admin
```

登录后可显示：

```plain
管理台
退出登录
```

---

## **五、上传 / 新建资产页表单优化**
### **当前问题**
表单只有 placeholder，没有 label。

用户输入后，字段含义会消失，体验不好。

### **调整要求**
所有表单项必须有明确 label。

例如：

```plain
产品名称
[请输入产品名称]

产品介绍
[请输入产品介绍]

产品链接
[请输入产品链接]

Figma链接（可选）
[请输入Figma链接]
```

---

## **六、表单字段结构**
### **Vibe Product 新增字段**
+ 产品名称
+ 产品介绍
+ 产品链接
+ 封面图链接，可选
+ 标签，可选
+ 排序值，可选

---

### **组件规范新增字段**
+ 组件名称
+ 组件介绍
+ 规范链接
+ Figma链接，可选
+ 标签，可选
+ 排序值，可选

---

## **七、上传页信息架构调整**
### **当前问题**
上传页左侧是表单，右侧又展示已有资产列表，信息过多且干扰当前任务。

### **调整要求**
上传页只保留“新增 / 编辑表单”。

不要在上传页旁边展示已有资产列表。

已有资产管理应放在管理列表页中。

---

## **八、管理页结构调整**
管理页建议拆成两个区域：

```plain
管理页 /admin

顶部：
- 新建 Vibe Product
- 新建组件规范
- 退出登录

内容：
- Vibe Product 管理列表
- 组件规范管理列表
```

---

## **九、管理列表操作**
登录后的管理列表中，每条资产需要支持：

+ 查看
+ 编辑
+ 删除

删除操作必须二次确认：

```plain
确认删除「Design Recorder」吗？
删除后不可恢复。
```

第一期可以直接从本地 mock / localStorage 中删除。

后续预留真实 API 删除能力。

---

## **十、列表页与管理页职责区分**
### **普通列表页**
例如：

```plain
/vibe-product
/components
```

只面向普通用户浏览。

操作：

+ 查看
+ 打开产品
+ 打开规范

不出现删除、编辑。

---

### **管理列表页**
例如：

```plain
/admin
```

面向管理员。

操作：

+ 新增
+ 编辑
+ 删除
+ 查看

---

## **十一、发布资产按钮调整**
当前「发布资产」容易让用户误以为普通用户可直接发布。

建议：

未登录时：

```plain
管理入口
```

登录后：

```plain
新建资产
```

或者：

```plain
进入管理台
```

---

## **十二、按钮与表单样式统一**
当前项目已走“硬朗、杂志式、设计机构官网”方向。

管理页也要保持一致。

### **需要避免**
+ 胶囊按钮
+ 大圆角输入框
+ SaaS后台风
+ 默认 Shadcn 表单感

### **推荐**
+ 直角或 2px 小圆角
+ 细线边框
+ 清晰 label
+ 大标题
+ 大留白
+ 分区线
+ 少阴影

---

## **十三、编辑能力**
点击「编辑」后进入：

```plain
/admin/edit/[id]
```

或打开同一个表单页面，并回填数据。

编辑页要求：

+ 表单 label 完整
+ 原数据自动回填
+ 保存后返回管理列表
+ 保存成功提示

---

## **十四、数据存储调整**
当前第一期可以继续使用本地数据。

但需要支持新增、编辑、删除后的持久化。

建议用：

```plain
localStorage
```

模拟。

并保留 service 抽象：

```plain
assetService.createAsset()
assetService.updateAsset()
assetService.deleteAsset()
assetService.getAssets()
```

页面不要直接操作 localStorage。

后期接服务器时，只需要替换 service。

---

## **十五、错误与校验**
新增 / 编辑表单需要基础校验：

### **必填**
+ 名称
+ 介绍
+ 链接

### **链接格式**
+ 产品链接必须是有效 URL
+ 规范链接必须是有效 URL
+ Figma链接如果填写，也必须是有效 URL

错误提示显示在字段下方。

---

## **十六、空状态**
如果某个分类暂无内容，需要显示空状态。

例如：

```plain
暂无资产
点击「新建资产」开始添加
```

不要让页面空白。


## **十八、后端预留要求**
所有登录、增删改查都必须通过 service 层封装。

当前可以 mock：

```plain
authService.login()
authService.logout()
authService.getCurrentUser()

assetService.createAsset()
assetService.updateAsset()
assetService.deleteAsset()
```

未来真实后端接口可替换为：

```plain
POST /api/auth/login
POST /api/assets
PUT /api/assets/:id
DELETE /api/assets/:id
GET /api/assets
```

## **二十、最终目标**
让管理流程从当前的：

```plain
页面里输入管理员密码 → 直接新增
```

调整为：

```plain
登录管理账号 → 进入管理台 → 新增 / 编辑 / 删除资产
```

普通用户只负责浏览和打开资产。

管理员才负责维护资产。

同时保持当前设计机构官网风格，不要退回传统后台。
