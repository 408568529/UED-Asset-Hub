00 / 基线

代码与数据必须分开

代码目录

Git 只管理这里的代码。

D:\UED-Asset-Hub-Host\UED-Asset-Hub\

真实数据目录

上传文件、资产记录和历史数据保存在仓库外。

D:\UED-Asset-Hub\runtime-data\
只要 DATA_DIR 指向仓库外的 runtime-data，更新代码不会覆盖真实业务数据。
不要执行 git reset --hard 或 git checkout -- .。它们会直接丢弃主机中未提交的修改。
01 / 本机

本机开发完成后上传 Git

单行执行

检查改动

确认没有真实数据、.env.local 或密钥文件。

git status
单行执行

创建功能分支

将 feature-name 改为本次功能名。

git switch -c codex/feature-name
单行执行

类型检查

npm run typecheck
单行执行

生产构建

npm run build

提交并推送

按下面从上到下执行；每一条都是独立命令。

第 1 条 / 暂存所有确认过的改动
git add -A
第 2 条 / 创建提交
git commit -m "Describe this release"
第 3 条 / 推送当前功能分支
git push -u origin HEAD

在 GitHub 创建 Pull Request，确认 base 为 main 后合并。

02 / 一次配置

Windows 主机改用 SSH 拉取

单行执行

生成主机密钥

ssh-keygen -t ed25519 -C "UED Asset Hub Windows Host"

按默认路径保存。绝不上传没有 .pub 后缀的私钥。

单行执行

复制公钥

Get-Content "$env:USERPROFILE\.ssh\id_ed25519.pub" | Set-Clipboard

在 GitHub 个人 SSH and GPG keys 新建 Authentication Key，再按 Ctrl + V 粘贴。

单行执行

验证连接

ssh -T git@github.com

首次询问时输入 yes；看到 successfully authenticated 即成功。

切换仓库远端

按下面从上到下执行；每一条都是独立命令。

第 1 条 / 进入主机代码目录
cd "D:\UED-Asset-Hub-Host\UED-Asset-Hub"
第 2 条 / 指定 Windows SSH 程序
git config --global core.sshCommand "C:/Windows/System32/OpenSSH/ssh.exe"
第 3 条 / 改为 SSH 远端地址
git remote set-url origin git@github.com:408568529/UED-Asset-Hub.git
第 4 条 / 确认地址
git remote -v
03 / 每次发布

更新主机代码并启动

先停止旧服务

在正在运行 npm run start 的 PowerShell 窗口按 Ctrl + C。

单行执行

进入目录

cd "D:\UED-Asset-Hub-Host\UED-Asset-Hub"
单行执行

检查主机状态

git status
条件执行

仅 package-lock 被修改时才保存

git stash push -m "host package lock before update" package-lock.json

如果 git status 显示工作区干净，直接跳过本步骤。若出现其他文件修改，先不要拉取或清理。

单行执行

拉取 main

git pull origin main
可选排错

仅构建异常或页面仍是旧版时清缓存

Remove-Item -Recurse -Force .\.next

正常更新可以跳过。执行后再重新运行 npm run build。

单行执行

严格重装依赖

npm ci

自动重建 node_modules，不改写 package-lock。

单行执行

构建

npm run build
单行执行

启动新版本

npm run start -- -H 0.0.0.0 -p 3027

该命令会持续运行。局域网访问：http://主机IP:3027。

04 / 临时演示

生成公网演示链接

Quick Tunnel 会把当前主机的站点公开到互联网。仅用于临时演示；确认没有展示敏感内容，并确保后台密码不是默认值。关闭隧道窗口后，链接立即失效。
当前主机

确认 cloudflared 安装位置

当前已验证跑通的程序路径：

C:\Program Files (x86)\cloudflared\cloudflared.exe
单行执行

确认工具可用

& "C:\Program Files (x86)\cloudflared\cloudflared.exe" --version
前提

先启动平台

在第一个 PowerShell 窗口启动 UED Asset Hub，并保持窗口不要关闭：

npm run start -- -H 0.0.0.0 -p 3027
单行执行

新开第二个 PowerShell 窗口，启动公网隧道

& "C:\Program Files (x86)\cloudflared\cloudflared.exe" tunnel --url http://127.0.0.1:3027

终端会打印一个 https://随机名称.trycloudflare.com 链接，复制该链接分享给外部人员。

结束演示

关闭公网链接

在运行 cloudflared 的第二个窗口按 Ctrl + C。该临时链接会立即停止，下一次启动会生成新的链接。

05 / 推荐

重要版本前备份数据

单行执行

代码与数据已分离，但重要发布前仍建议压缩备份一次。

$ts = Get-Date -Format "yyyyMMdd-HHmmss"; Compress-Archive -Path "D:\UED-Asset-Hub\runtime-data\*" -DestinationPath "D:\UED-Asset-Hub\runtime-data-backup-$ts.zip"
06 / 排错

常见问题

Git HTTPS / TLS 报错

主机已改用 SSH。先检查 SSH：

ssh -T git@github.com

成功后执行 git pull origin main，不需要 Token。

Next SWC 不是有效 Win32 应用

执行本页的 npm ci，它会重装与 Windows 匹配的原生依赖。
